# dropkit / examples

This directory holds sample inputs and quickstart artifacts.

## What's here

- `diagram.png` — generated by `bash quickstart.sh` from the repo root. A
  small architecture-style diagram used to exercise the file-to-markdown
  image pipeline without needing an LLM or a real document.
- `processing/` — output of the `split_image.py overview` run during
  the quickstart. Gitignored. Safe to delete; rerunning `quickstart.sh`
  regenerates it.

## Bring your own

The skills are designed for real inputs you already have:

| Skill | Try it on |
|---|---|
| `file-to-markdown` (documents) | any PDF, DOCX, PPTX, XLSX, XLS you have on disk |
| `file-to-markdown` (images) | a screenshot of a whiteboard, a Lucidchart export, an event-storming photo |
| `markdown-to-html` | any `.md` file |
| `msg-to-markdown` | an Outlook `.msg` export |
| `jira` / `jira-align` / `confluence-crawler` | your real Atlassian instance, after `setup_credentials.sh` |

The skill `SKILL.md` files describe the prompt patterns each skill responds to.
